## 概览：3 层、7 条主线

```
ESP32-C5 ×3  ──UDP:5005──>  RZ/G2L (Rust)  ──WebSocket:8765──>  浏览器
                                 │
                                 └── HTTP:8080 (静态UI + REST API)
```

## 第一层：ESP32-C5 感知节点 → UDP 上行

WiFi CSI 帧通过 ESP32-C5 的 promiscuous 模式采集，经片上预处理后通过 UDP 发送到 RZ/G2L 主控。

### 数据采集

```
WiFi CSI 帧 (HE40, 242子载波, 2.4/5GHz双频)
  │
  ├─ csi_collector.c: wifi_promiscuous_rx_cb()
  │    ├─ 信道跳转: 6ch × 50ms dwell (NVS 配置优先, 硬编码兜底)
  │    ├─ 速率限制: ≥50ms 发送间隔, 防止 lwIP pbuf 耗尽
  │    ├─ 互斥保护: FreeRTOS Mutex 序列化 esp_wifi_set_channel()
  │    └─ UDP sendto() → RZ/G2L:5005
  │
  ├─ edge_processing.c (片上边缘计算)
  │    ├─ Biquad IIR 带通滤波 (呼吸 0.1-0.5Hz, 心率 0.8-2.0Hz)
  │    ├─ NaN/Inf 防护 + 参数范围校验 (fs>0, f_lo<f_hi, Nyquist)
  │    ├─ 相位提取 + 展开 + 运动能量 + 存在检测
  │    └─ 打包为 ADR-039 边缘生命体征包 (magic 0xC511_0002)
  │
  └─ 3 种上行包类型:
       ├─ CSI 原始帧      (0xC511_0001)  ← 主力数据, 20字节头 + 振幅/相位
       ├─ 边缘生命体征     (0xC511_0002)  ← 低带宽备选, 心率/呼吸/运动/存在
       └─ WASM 边缘事件   (0xC511_0005)  ← 19个边缘模块 (步态/癫痫/睡眠等)
```

### 容错机制

| 机制 | 位置 | 说明 |
|------|------|------|
| UDP 发送失败重启 | main.c:208 | sendto init 失败 → esp_restart() |
| WiFi 断线重连 | main.c:78-96 | 指数退避 500ms×retry, 永久失败检测 6 种 reason |
| 密码 null 终止 | main.c:137 | strncpy 后显式 `\0` |
| 环形缓冲区溢出 | csi_collector.c:283 | 4096 次丢弃记一次 warning 日志 |
| 信号量超时 | csi_collector.c:438 | 100ms 超时 → warning, 跳过一次跳转 |

## 第二层：Rust 服务端 — 每帧处理管线

服务端由 `udp_receiver_task` 驱动。每收到一个 UDP CSI 帧，按顺序执行 11 个处理步骤。

### 阶段 1：每节点独立管线 (ns 作用域, 持有 s.node_states 借用)

```
frame.amplitudes + frame.phases
  │
  ├─ [1] SignalPipeline.process()              ← wifi_densepose_signal crate
  │       PhaseSanitizer(相位展开+异常值移除) → HardwareNormalizer
  │       → HampelFilter(中值滤波) → MotionDetector(自适应阈值)
  │       → CoherenceState + GatePolicy(信号质量门控)
  │       产出: SignalPipelineOutput { motion_score, human_detected,
  │              gate_allows_update, coherence_score, ... }
  │       消费: motion_score → 运动分类 | cleaned_amplitudes → VitalsBridge
  │
  ├─ [2] extract_features_from_frame()         ← 手写信号处理
  │       四维特征: 帧间差(0.4) + 方差(0.2) + 频带功率(0.25) + 变化点(0.15)
  │       产出: FeatureInfo { mean_rssi, variance, spectral_power,
  │              dominant_freq_hz, motion_band_power, breathing_band_power, ... }
  │            + raw_motion (LLM趋势+边缘引擎) + breathing_rate_hz + sub_variances
  │
  ├─ [3] 动态采样率测量
  │       帧到达间隔 dt → 瞬时频率 1/dt → EMA α=0.15 平滑 → ns.measured_sample_rate
  │       替代硬编码 20Hz, 自适应 ESP32-C5 实际发送速率
  │
  └─ [4] 运动分类 (从 signal_pipeline, ADR-142)
           signal_out.motion_score → EMA α=0.15 → 阈值分类:
           >0.15 "active" | >0.08 "present_moving" | >0.03 "present_still" | else "absent"
```

### 阶段 2：跨节点全局处理 (ns 已释放, s 完整可用)

```
  ├─ [5] VitalsBridge.extract()                ← wifi_densepose_vitals (上游算法)
  │      算法: 与 wifi_densepose_wifiscan::CoarseBreathingExtractor 相同
  │      EMA预处理(静态分量抑制) → IIR带通滤波 → 零交叉计数(呼吸) + 自相关峰值(心率)
  │      输入: signal_pipeline 清理后的振幅 + 相位 (如果可用; 否则用原始数据)
  │      窗口: 呼吸30s / 心率15s (可配置)
  │      产出: breathing_rate_bpm, heart_rate_bpm, br_confidence, hr_confidence
  │
  ├─ [6] CIRBridge.process()                   ← ISTA 稀疏信道脉冲响应估计
  │      产出: cir_distance_m (ToF 飞行时间测距, 用于三角定位)
  │
  ├─ [7] FieldBridge.feed()                    ← 电磁场物理模型
  │      前 AUTO_CALIBRATION_FRAMES=600 帧 (~30s@20Hz): 空房间基线学习
  │      校准完成后: 每帧输出 perturbation.energy → 注入 signal_field.values
  │      消费: signal_field 热力图 → 浏览器地图渲染
  │
  ├─ [8] TriageEngine.process()                ← START 分诊引擎
  │      输入: VitalSignsInput { BR, HR, 置信度, motion_score, RSSI, signal_quality }
  │      输出: 5 级分诊 — Immediate(红) / Delayed(黄) / Minor(绿) / Deceased(黑) / Unknown(灰)
  │      产出: TriageUpdate { survivors: [{id, triage, position, position_confidence}] }
  │
  ├─ [9] EdgeModuleEngine.process_frame()      ← 19 个边缘模块
  │      模块: gait, loitering, vibration, ltl, meta_adapt, sparse_recovery,
  │            vital_trend, attractor, coherence, resp_distress, confined_space,
  │            panic_motion, sleep_apnea, cardiac, seizure, intrusion, occupancy,
  │            mincut, weapon_detect
  │      产出: Vec<EdgeAlert> { module, event_name, severity, value }
  │
  ├─ [10] LocalizationBridge + TrackingBridge  ← 多节点定位 + Kalman 追踪
  │       LocalizationBridge: 3+ 节点 RSSI + CIR ToF → 最小二乘三角定位
  │       TrackingBridge: Kalman 6-D CV 模型 + 8维生物特征 embedding 重识别
  │       lost_pool 5min → 三级匹配 (person_id → Re-ID → 新建)
  │       产出: TrackedSurvivor { position, velocity, reidentified, confidence }
  │
  └─ [11] AlertingBridge.generate_alert()      ← 结构化告警生命周期
           触发条件: triage ∈ {Immediate, Delayed, Deceased}
           产出: AlertInfo { id, survivor_id, title, message, priority, status }
           入队: generated_alerts (FIFO, 最大50条)
```

### 阶段 3：组装 WebSocket 推送

```
SensingUpdate {
    type:           "sensing_update"
    timestamp:      f64             ← Unix 时间戳
    source:         "esp32"
    tick:           u64             ← 帧序号
    nodes:          Vec<NodeInfo>   ← 所有活跃节点快照 (RSSI/BR/HR/位置/振幅)
    features:       FeatureInfo     ← 信号特征 (7 个维度)
    classification: ClassificationInfo ← 运动级别 + 存在检测 + 置信度
    signal_field:   SignalField     ← 3D 信号场热力图 (含 field_perturbation)
    vital_signs:    Option<VitalSigns>      ← 呼吸/心率/置信度/信号质量
    triage_update:  Option<TriageUpdate>    ← START 分诊 + 伤员列表
    wasm_alerts:    Option<Vec<EdgeAlert>>  ← 边缘模块告警
    tracked_survivors: Option<Vec<TrackedSurvivor>> ← Kalman 追踪位置
    pose_keypoints: Option<Vec<[f64;4]>>     ← 合成 3D 骨架
    persons:        Option<Vec<PersonDetection>> ← 人员检测 (COCO 17 关键点)
    estimated_persons: Option<usize>         ← 人数估计
    alerts:         Option<Vec<Value>>       ← 告警 drain (逐个推送)
}
  │
  └─ s.tx.send(json) → tokio::sync::broadcast channel (容量 1024)
```

### 阶段 4：后台消费任务

| 任务 | 周期 | 功能 |
|------|:---:|------|
| broadcast_tick_task | 500ms | 排空 alerting_bridge.drain_alerts() → 逐个推 alert JSON; 重播 latest_update |
| periodic_agent_task | 5s | MedicalAgent.analyze() → 云 LLM → agent_analysis JSON |
| simulated_data_task | tick_ms | (仅模拟模式) 生成假 CSI 数据驱动管线 |

## 第三层：浏览器 UI — WebSocket 消息路由

```
WebSocket onmessage → JSON.type 路由:
  │
  ├─ "sensing_update" → handleUpdate()
  │    ├─ renderSurvivorList()        ← triage_update.survivors (按严重度排序)
  │    ├─ drawMap()                   ← signal_field.values + _trackedPositions
  │    │     physToCanvas() 坐标映射, 节点标记, 热力图叠加
  │    ├─ drawSparklines()            ← vital_signs (呼吸/心率 30s 趋势图)
  │    ├─ renderWasmAlerts()          ← wasm_alerts[0..12] (19 模块动态计数)
  │    ├─ animate() skeleton          ← pose_keypoints / persons (合成 + 实时)
  │    │     synthetic/estimated 标签 (数据源指示器)
  │    └─ 合并 _trackedPositions      ← tracked_survivors (Kalman 平滑位置优先)
  │
  ├─ "alert" → handleAlertMessage()
  │    └─ 渲染告警侧栏, 按 priority 着色
  │       Critical→红, High→橙, Medium→黄, Low→蓝
  │
  ├─ "agent_analysis" → handleAgentStream()
  │    └─ LLM 分析结果流式展示 (50条上限, 自动清理)
  │
  ├─ "edge_vitals" → handleEdgeVitals()
  │    └─ 节点生命体征面板更新
  │
  ├─ "ping" / "pong" ← 心跳
  │
  └─ "patient_register" ← 上行: 注册伤员信息到 LLM 引擎
```
