## 三、固件层 — ESP32-C5 CSI 感知节点

**位置**：`firmware/esp32-c5-csi-node/`（76 个文件）

### 3.1 CSI 采集（`csi_collector.c/h`）

**ADR-018 帧格式**（20 字节头 + I/Q 数据）：

| 偏移 | 大小 | 字段 |
|------|------|------|
| 0–3 | 4B | Magic `0xC5110001` (LE) |
| 4 | 1B | Node ID |
| 5 | 1B | 天线数 |
| 6–7 | 2B | 子载波数 (u16 LE) |
| 8–11 | 4B | 频率 MHz (u32 LE) |
| 12–15 | 4B | 序列号 (u32 LE) |
| 16 | 1B | RSSI (i8) |
| 17 | 1B | 噪声底 (i8) |
| 18–19 | 2B | 保留 |
| 20+ | NB | 原始 I/Q 数据 |

**关键特性**：
- 最大帧 4116 字节（适配 WiFi 6 484 子载波）
- 50Hz 发送限速（防 lwIP pbuf 耗尽）
- MAC 地址过滤（ADR-060）
- `first_word_invalid` 跳过（C5/C6 专有）
- 6GHz WiFi 6E 频段支持
- 通道跳跃（ADR-029）：6 信道轮转

### 3.2 边缘计算（`edge_processing.c/h`）

**双核 DSP 架构**：
- Core 0（生产者）：CSI 回调 → SPSC 环形缓冲（16 槽）
- Core 1（消费者）：优先级 5，8KB 栈，14 步处理管线

**14 步处理管线**：
1. 相位提取（atan2）
2. 相位解卷绕
3. Welford 在线方差
4. Top-K 子载波选择（插入排序）
5. 主相位追踪
6. IIR 带通：呼吸 0.1–0.5Hz / 心率 0.8–2.0Hz
7. 过零 BPM 估计（≥64 帧）
8. 运动能量（20 帧窗口方差）
9. 存在检测（1200 帧自适应基线，mean+3σ）
10. 跌倒检测（相位加速度 + 3 帧消抖 + 5s 冷却）
11. 多人生命体征（子载波群聚类）
12. 增量压缩（XOR + RLE）
13. Vitals 包（magic `0xC5110002`）
14. WASM 分发

**处理层级**：Tier 0（原始透传）/ Tier 1（基础 DSP）/ Tier 2（完整处理）

### 3.3 WASM 运行时（`wasm_runtime.c/h`）

- WASM3 解释器，4 并发模块，160KB PSRAM/槽
- 12 个 Host API（csi_get_phase/amplitude/variance/bpm_breathing/bpm_heartrate/presence/motion/n_persons/timestamp/emit_event/log/phase_history）
- 能力位掩码访问控制
- 10ms/帧预算执行 + 超限自动停止
- UDP 输出包（magic `0xC5110004`）

### 3.4 其他固件模块

| 模块 | 功能 |
|------|------|
| `stream_sender.c` | UDP 发送 + ENOMEM 退避 |
| `rvf_parser.c` | RVF 二进制容器解析（Header+Manifest+WASM+签名） |
| `ota_update.c` | HTTP OTA 固件升级 |
| `power_mgmt.c` | 电池供电管理 |
| `mmwave_sensor.c` | 毫米波传感器 Kalman 融合（ADC-063） |
| `swarm_bridge.c` | 蜂群桥接（ADC-066） |
| `mock_csi.c` | 10 种测试场景，确定性 LFSR |
| `nvs_config.c` | NVS 运行时配置 |
| `display_*.c` | AMOLED 显示（ADR-045） |

### 4.2 主服务数据流

```
ESP32 UDP (port 5005, magic 0xC5110001)
  │
  ├─ parse_esp32_frame()        → 振幅 + 相位 + 元数据
  ├─ extract_features()         → 运动/存在检测 + 特征向量
  ├─ VitalSignDetector          → 呼吸率、心率、置信度、信号质量
  ├─ TriageEngine.process()     → START 分诊 + 伤员追踪 + 恶化检测 + 群体评估
  ├─ EdgeModuleEngine           → 19 个边缘模块告警
  ├─ 信号场生成                  → 20×20 网格热力图
  ├─ DensePose 骨架（可选）       → 17 点 COCO 关键点
  │
  └─ SensingUpdate → WebSocket /ws/sensing → triage.html
```

**SensingUpdate JSON 结构**（15 个字段）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | string | 固定 `"sensing_update"` |
| `timestamp` | number | Unix 时间戳 |
| `source` | string | `"simulated"` 或 `"esp32"` |
| `tick` | integer | 帧序号 |
| `nodes` | array | 节点信息（RSSI、位置、振幅） |
| `features` | object | CSI 信号特征 |
| `classification` | object | 运动/存在分类 |
| `signal_field` | object | 20×20 信号场热力图 |
| `vital_signs` | object? | 生命体征（呼吸率/心率） |
| `triage_update` | object? | MAT 分诊数据（核心） |
| `wasm_alerts` | array? | 边缘模块异常事件 |
| `pose_keypoints` | array? | DensePose 骨架 |
| `model_status` | object? | 模型加载状态 |
| `persons` | array? | 多人检测结果 |
| `estimated_persons` | integer? | 估计人数 |

## 五、MAT 分诊系统（核心竞赛功能）

**位置**：`wifi-densepose-mat` crate（43 文件）+ `sensing-server/src/mat_pipeline.rs`（~500 行）

### 5.1 START 协议分诊规则

| 等级 | 颜色 | 判定条件 |
|------|------|---------|
| **Immediate** | 🔴 红 | RR>30 或 RR<10，或 HR>120 或 HR<40 |
| **Delayed** | 🟡 黄 | RR 10–12 或 25–30，HR 40–50 或 100–120 |
| **Minor** | 🟢 绿 | 正常体征 + 可自主移动（motion > 0.3） |
| **Deceased** | ⚫ 黑 | 无生命体征 |
| **Unknown** | ⬜ 灰 | 信号质量 < 0.05 |

### 5.2 TriageEngine 功能

- **伤员匹配**：person_id + node_id 关联，5s 内同节点视为同一人
- **生命体征历史**：30 帧滑动窗口 EMA 平滑
- **多节点三角定位**：≥2 节点加权三角定位；1 节点 RSSI 距离估算
- **恶化检测**：分诊等级连续跳 ≥2 级 → DETERIORATION 告警
- **年龄估算**：呼吸率/心率 → Infant(<2y) / Child(2–12y) / Adult / Elderly(60y+)
- **群体伤情评估**：统计 total/immediate/delayed/minor/deceased + severity + rescuer_estimate（Immediate×4 + Delayed×2 + Minor/2）
- **自动过期**：超时未更新的伤员自动标记

### 5.3 MAT crate 完整架构（DDD 分层）

| 上下文 | 核心类型 | 功能 |
|--------|---------|------|
| **Detection** | BreathingDetector, HeartbeatDetector, EnsembleClassifier | 多模态体征检测 + 集成融合 |
| **Localization** | Triangulator, DepthEstimator, PositionFuser | 3D 伤员定位 |
| **Alerting** | AlertGenerator, TriageService, PriorityCalculator | START 协议告警分发 |
| **ML** | DebrisPenetrationModel, VitalSignsClassifier | 废墟穿透建模 + 体征分类 |
| **Tracking** | SurvivorTracker, KalmanState, CsiFingerprint | 多帧身份追踪 |
| **Domain** | Survivor, DisasterEvent, ScanZone, TriageStatus | 核心领域模型 + 事件溯源 |
| **API** | REST + WebSocket + DTOs | HTTP API 接口层 |

## 六、信号处理管线

**位置**：`wifi-densepose-signal` crate（28 .rs 文件）

| 模块 | 技术 | 输出 |
|------|------|------|
| **呼吸率检测** | Fresnel 模型 + FFT (0.1–0.5Hz) + 过零 BPM + Goertzel 置信度 | 6–30 BPM |
| **心率检测** | BVP 提取 + IIR 带通 (0.667–2.0Hz) + 相位方差 | 40–120 BPM |
| **运动检测** | 帧间振幅方差 + 相位方差 + 自适应阈值 + 5 帧消抖 | motion_score [0,1] |
| **存在检测** | 1200 帧自适应基线校准 + mean+3σ 阈值 | present/absent |
| **子载波选择** | Top-30 高方差子载波 | 提升 SNR |
| **多人分离** | MinCut 子载波分离 + 多人匹配 | ≥2 人独立追踪 |
| **姿态追踪** | 17 关键点 Kalman + DensePose | 3D 骨架 |
| **Hampel 滤波** | 异常值过滤 | 数据清洗 |
| **信号场生成** | 20×20 子载波方差 → 方向热点图 | 2D 热力图 |

### 性能优化

| 优化 | 说明 | 提升 |
|------|------|:--:|
| 原生 FPU | 替代 WASM `libm` 软浮点，使用硬件 `f32::sqrt()` | 5–10× |
| 单编译单元 | 所有模块内联到单一 struct，激进内联 + LTO | ~2× |
| 缓存友好 | 连续内存布局，减少 cache miss | ~1.5× |
| 零 FFI 开销 | 无跨 WASM 边界调用 | 消除延迟 |
