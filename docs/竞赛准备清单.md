### 2.1 项目简介（200 字 elevator pitch）

**建议在 `docs/项目简介.md` 写入以下内容：**

> **WCES — WiFi 穿墙生命感知系统**
>
> 本项目利用商用 WiFi 信号的信道状态信息（CSI），在无需摄像头、无穿戴设备、无互联网的条件下，
> 实现穿墙人体姿态重建与生命体征监测。系统由 3 个 ESP32-C5 WiFi 6 感知节点 + 瑞萨 RZ/V2H AI 主控组成，
> 总硬件成本仅 ¥1000。核心创新在于：
> 1. 利用 WiFi 6 的 484 子载波高分辨率 CSI，4× 传统方案感知精度
> 2. 65 个 WASM 热加载边缘 AI 模块，覆盖睡眠呼吸暂停、心律失常、跌倒检测等医疗场景
> 3. Rust 实现的 54,000 帧/秒处理管道，比 Python 快 810 倍
> 4. 完全本地运行，无隐私泄露风险
>
> 应用场景：灾害搜救（穿墙检测幸存者）、医院无接触监护、居家老人跌倒检测。

### 5.3 现场故障预案

```markdown
# 现场故障应急方案

## 场景 1：WiFi 干扰严重（展会现场常见）
→ 切换到有线以太网（备千兆网线 × 2）
→ 降低节点距离到 1 米
→ 换 5GHz 信道（避开拥堵 2.4GHz）

## 场景 2：某节点掉线
→ 2 节点仍可工作（性能降级，但系统不崩溃）
→ 快速排查：检查电源线 → 按复位键 → 重新烧录

## 场景 3：RZ/V2H 死机
→ 备一个树莓派 4B 或笔记本电脑跑 Rust 服务端（armeabi 交叉编译版本）
→ 或者：ESP32-C5 节点独立运行边缘模块（不依赖主控）

## 场景 4：屏幕不亮
→ 备 HDMI 线直连评委的显示器/投影仪
→ 或者：评委手机连 WiFi 访问 Web UI (http://<IP>:8080)

## 场景 5：人体感知效果不好
→ 让志愿者穿金属饰品的衣物（增强反射）
→ 志愿者做大动作（挥手、走路）而非微动
→ 降低演示距离到 1-2 米
```
