# 竞赛需求 vs 项目现有能力 — 差距分析

> 比赛：全国大学生嵌入式芯片与系统设计竞赛（瑞萨赛道）
> 项目：WCES — 基于 WiFi 6 CSI 的极端条件人员监护系统
> 硬件：瑞萨 RZ/V2H + 3× ESP32-C5

## 二、需要优先开发的 3 个核心模块
### 模块 1：分诊仪表盘 UI（Triage Dashboard）
**用途：** 竞赛现场展示的核心屏幕，评委直接看到

**功能需求：**
- 2D 俯视地图，标注 3 个 C5 节点位置
- 检测到的伤员用圆点表示，颜色 = START 分诊状态
- 点击伤员显示详情：呼吸率、心率、运动状态、分诊等级、位置坐标
- 顶部统计栏：总计 / Immediate / Delayed / Minor / Deceased
- 告警列表（时间倒序）
- 资源需求估计

**技术方案：**
```
方案 A（推荐）：在现有 Web UI 添加分诊面板
  - 复用 sensing-server 的 WebSocket 8765
  - 新增 /api/triage/survivors 接口
  - 前端用 Three.js 2D Canvas 渲染伤员地图

方案 B：Python 独立仪表板
  - FastAPI + WebSocket 客户端
  - Matplotlib/Plotly 渲染
```

### 模块 2：MAT 管道集成（Data Pipeline）
**用途：** 确保 CSI 数据流入 MAT 系统

**现有状态检查：**

`wifi-densepose-mat` crate 已实现完整的 domain + alerting + localization + tracking 层。
`wifi-densepose-sensing-server` 的 `main.rs` 需要确认是否调用了 MAT。

**需要验证/实现的：**

```rust
// sensing-server 主循环中 MAT 集成（伪代码）
fn csi_frame_loop() {
    let csi_frame = receiver.recv().unwrap();
    
    // 1. 信号处理（已有）
    let features = signal_pipeline.process(&csi_frame);
    
    // 2. 人员检测 + 生命体征（已有）
    let person = motion_detector.detect(&features);
    let vitals = vitals_extractor.extract(&features);
    
    // 3. MAT 分诊（需确认/连接）⭐
    let triage = TriageService::calculate_triage(&vitals);
    
    // 4. 伤员追踪（需确认/连接）⭐
    for survivor in &mut survivors {
        survivor.update_vitals(&vitals);
        if TriageService::should_upgrade(survivor) {
            survivor.upgrade_triage();
            alert_system.push(survivor);
        }
    }
    
    // 5. 推送 UI（需新增）⭐
    ws_broadcast(&TriageDashboardUpdate {
        survivors: &survivors,
        assessment: TriageService::evaluate_mass_casualty(&survivors),
    });
}
```

### 模块 3：系统集成测试
**用途：** 保证演示时不出问题
