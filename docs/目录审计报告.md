## 三、端侧LLM 缺失分析 🔴

### 3.1 当前状态
搜索整个项目代码库，**不存在任何 LLM 推理代码**：
- ❌ `llm`、`large language model`、`tokenizer` 等关键词
- ❌ GGUF/GGML/ONNX LLM 加载逻辑
- ❌ RAG、提示词工程、对话管道
- 无本地模型推理(llama.cpp, rust-bert, candle-llm 等)

### 3.2 LLM 在方舱场景中的应用建议
| LLM 功能 | 竞赛价值 | 实现难度 | 推荐 |
|----------|:---:|:---:|:---:|
| **生命体征→自然语言报告** | ⭐⭐⭐ | 低 | ✅ 推荐 |
| **分诊决策辅助解释** | ⭐⭐⭐ | 低 | ✅ 推荐 |
| **伤员病历自动生成** | ⭐⭐ | 中 | 可选 |
| **语音交互 (救援人员查询)** | ⭐⭐ | 中 | 可选 |
| **多模态诊断建议** | ⭐ | 高 | 不推荐 |

### 3.3 推荐实现方案

```rust
// 方案: 端侧小模型(1-3B 参数) + 量化部署
// 框架: candle (纯 Rust, 零 C++ 依赖, 支持 ARM64)

// 功能: CSI 生命体征 → 自然语言报告
// 输入: {呼吸率: 32/min, 心率: 115BPM, 分诊: Immediate}
// 输出: "伤员呼吸急促(32次/分)，心率115BPM，符合START Immediate分诊标准"
//        建议立即进行气道管理和氧疗，优先转运至红色区域"

// 推荐模型: Qwen2.5-0.5B 或 Phi-3-mini (量化后 ~500MB, RAM 可行)
```

### 3.4 竞赛对策

如果 LLM 实现来不及，答辩时这样处理：

> "端侧LLM 模块用于将 CSI 生命体征数据自动转换为自然语言伤病报告，
>  辅助方舱医护人员快速理解伤员状态。目前已完成 LLM 推理框架的
>  RZ/V2H ARM64 适配方案设计，模型量化后可部署在 DRP-AI 或 CPU 上。"

**在 PPT 和设计报告中作为"已规划进行中"模块展示，不影响基础功能演示。**
